# ZhuTu ZhuJi · 铸图铸机

**A human-AI co-creation system running inside Claude Code as an MCP plugin.
The planning layer forges architecture blueprints through spiral dialogue.
The execution layer enforces them with deterministic precision.**

---

## What Makes This Different

- **The Thought Partner Model** — Answers do not exist before the conversation.
  The Mentor expands the possibility space; the human finds new directions;
  architecture accumulates in the spiral between them.

- **A One-Way Contract Between Layers** — ZhuTu's signed blueprint JSON is
  ZhuJi's only source of truth. The execution layer never rewrites the planning
  layer's conclusions. It can only escalate genuine architectural crises back
  to the human.

- **Determinism Returned to Code** — AST-based patch fusion, content_hash
  verification, dependency graph propagation, and invariant validation are all
  handled by deterministic Python. Models own reasoning. Scripts own everything
  else. The boundary is structural, not stylistic.

- **Role Names Are Behavioral Contracts** — Mentor opens genuine collaboration;
  Explainer closes it. Cognitive separation across roles is achieved through
  temperature and extended thinking configuration — not by switching models.

- **Cache as Economic Foundation** — Append-only session discipline keeps cache
  prefixes stable. One Opus Fusion call at session end synthesizes all raw
  materials into the next session's cold-start document — compounding forward
  as a multiplier on all future work.
